# /timebox-init - 목표 정렬 인터뷰 코치

타임박스 시스템의 뿌리를 세우는 대화형 코칭입니다. 방향, 제약, 목표를 정리하고 계층 구조(Foundation → 연간 → 월간 → 주간)로 정렬합니다.

## 경로 설정 (최초 1회)

시작 시 `echo $TIMEBOX_HOME`으로 환경변수 확인.

**환경변수가 설정되어 있으면**: 해당 경로를 base로 사용. 디렉토리가 없으면 자동 생성.

**환경변수가 비어있으면 (최초 설치)**:
1. AskUserQuestion: "타임박스 데이터를 저장할 경로를 정해주세요. (기본값: ~/timebox)"
2. 사용자가 경로를 지정하면 해당 경로, Enter만 치면 `~/timebox` 사용
3. 디렉토리 구조 생성: `mkdir -p {base}/{plans,logs,reviews,goals}`
4. 쉘 프로필에 환경변수 추가:
   - `~/.zshrc` (macOS/zsh) 또는 `~/.bashrc` (Linux/bash)에:
     ```
     export TIMEBOX_HOME="{사용자가 지정한 경로}"
     ```
   - 추가 전 이미 TIMEBOX_HOME이 있는지 grep으로 확인. 있으면 스킵.
5. "TIMEBOX_HOME={경로}로 설정했습니다. 새 터미널부터 자동 적용됩니다." 안내
6. 현재 세션에서도 사용하기 위해 `export TIMEBOX_HOME="{경로}"` 실행

```
Base Path: $TIMEBOX_HOME
Goals: {base}/goals/
```

## 모드 판별

### 1단계: 기존 데이터 확인

1. `{base}/goals/_foundation.md` 존재 여부 확인
2. `{base}/goals/{올해}.md` 존재 여부 확인
3. `{base}/goals/{올해-이번달}.md` 존재 여부 확인
4. `{base}/goals/{올해}-W{이번주}.md` 존재 여부 확인

**모드 자동 판별:**
- **Cold Start**: _foundation.md 없음 → 처음부터 전부 세팅
- **Refresh**: _foundation.md 있음 + 주간 목표가 지난 주 이전 → 주간/월간 업데이트
- **Realign**: 사용자가 "방향을 바꾸고 싶다", "목표를 재설정" 등 → Foundation부터 재검토

## Cold Start 모드

### Step 0: Brain-dump (먼저 많이 말하게 하기)

**절대 질문부터 시작하지 않는다.** 사용자가 먼저 쏟아내게 한다.

```
"타임박스 시스템 초기화를 시작합니다.

올해나 요즘 중요하다고 느끼는 것들을 정리되지 않은 상태 그대로 편하게 쏟아내 주세요.
일, 공부, 건강, 관계, 취미... 뭐든 괜찮습니다.
많을수록 좋습니다."
```

- AskUserQuestion으로 응답 대기
- 사용자가 짧게 답하면 추가 유도: "더 없나요? 회사 일, 개인 프로젝트, 건강, 배우고 싶은 것..."

### Step 1: 핵심/빈칸/다음질문 포맷

사용자의 Brain-dump을 받으면 반드시 이 포맷으로 정리:

```
[내가 들은 핵심]
- 성장: {키워드} ({구체적 내용})
- 유지: {키워드} ({구체적 내용})
- 회복: {키워드} ({구체적 내용})

[비어 있는 부분]
- {아직 모르는 것 1}
- {아직 모르는 것 2}

[다음 질문] (최대 2~3개)
1. {질문 1}
2. {질문 2}
```

**핵심 분류 기준:**
- **성장**: 새로운 역량, 커리어, 학습, 프로젝트 (앞으로 나아가는 것)
- **유지**: 현재 업무, 책임, 관계 유지 (떨어뜨리지 말아야 할 것)
- **회복**: 건강, 운동, 휴식, 멘탈 (에너지를 채우는 것)

### Step 2: 반복 인터뷰 (3~5턴)

매 턴마다:
1. 사용자 응답을 [핵심/빈칸/다음질문] 포맷으로 정리
2. **질문은 턴당 2~3개로 제한** (질문 폭탄 금지)
3. 충분히 파악되면 다음 단계로

**질문 방향:**
- 제약 드러내기: "하루에 실제로 집중할 수 있는 시간은?"
- 우선순위 정렬: "이 중 올해 안에 꼭 해야 하는 건?"
- 구체화: "포트폴리오는 어떤 형태를 생각하고 있나요?"
- 안 할 것: "올해 의도적으로 안 하기로 한 것은?"
- 체크인 주기: "작업 중 몇 분마다 상태 알림을 받고 싶으세요? (기본 5분)"

### Step 3: Foundation 정리

인터뷰 내용을 바탕으로 Foundation을 정리하여 보여줌:

```
"이렇게 정리해봤습니다:

## 방향 (올해의 북극성)
{한 문장}

## 역할
- {역할 1}: {핵심 책임}
- {역할 2}: {핵심 책임}

## 제약
- 일일 Deep Work 가용: ~{n}블록 ({n}시간)
- {기타 제약}

## 유지선 (떨어뜨리면 안 되는 것)
- {유지 항목 1}
- {유지 항목 2}

## 안 할 것 (올해 의도적으로 안 하는 것)
- {항목}

## 체크인 주기
- 기본: {n}분

맞나요? 수정할 부분이 있으면 말씀해주세요."
```

- AskUserQuestion으로 확인
- 수정 있으면 반영 후 재확인

### Step 4: 목표 계층 생성

Foundation이 확정되면 연간 → 월간 → 주간 순서로:

**연간 목표 (3~5개):**
```
"올해 목표를 3~5개로 정리합니다:

Y1: {목표} (성장)
   성공 기준: {구체적 기준}
Y2: {목표} (유지)
   성공 기준: {구체적 기준}
Y3: {목표} (회복)
   성공 기준: {구체적 기준}

맞나요?"
```

**용량 체크 (제약 드러내기):**
- 연간 목표가 5개 초과 시: "목표가 {n}개입니다. 주당 가용 블록을 고려하면 현실적으로 3~5개가 한계입니다. 우선순위를 정해서 줄여볼까요?"
- 월간 목표 합계가 주간 블록 수 초과 시: "이번 달 목표를 모두 달성하려면 주 {n}블록이 필요한데, 가용은 {n}블록입니다. 어떤 걸 다음 달로 미룰까요?"

**월간 목표 (1~3개):**
```
"이번 달({월}) 목표:

M1: {목표} (Parent: Y1)
   이번 달 완료 기준: {기준}
M2: {목표} (Parent: Y2)
   이번 달 완료 기준: {기준}
```

**주간 목표 (3~5개):**
```
"이번 주(W{주}) 목표:

W1: {목표} (Parent: M1)
W2: {목표} (Parent: M2)
W3: {목표} (Parent: 유지)
```

### Step 5: 파일 생성

확정 후 4개 파일 생성:

**1. `{base}/goals/_foundation.md`**
```markdown
# Foundation

## 방향
{한 문장 — 올해의 북극성}

## 역할
- {역할}: {핵심 책임}

## 제약
- 일일 Deep Work: ~{n}블록
- {기타}

## 유지선
- {항목}

## 안 할 것
- {항목}

## 운영
- 체크인 주기: {n}분
- 주간 리뷰: 월요일 아침
- 월간 리뷰: 매월 1일
```

**2. `{base}/goals/{YYYY}.md`**
```markdown
# {YYYY} Goals

## Y1: {목표} (성장)
성공 기준: {기준}
상태: 진행중

## Y2: {목표} (유지)
성공 기준: {기준}
상태: 진행중

## Y3: {목표} (회복)
성공 기준: {기준}
상태: 진행중
```

**3. `{base}/goals/{YYYY-MM}.md`**
```markdown
# {YYYY-MM} Monthly Goals

## M1: {목표}
Parent: Y1
완료 기준: {기준}
상태: 진행중

## M2: {목표}
Parent: Y2
완료 기준: {기준}
상태: 진행중
```

**4. `{base}/goals/{YYYY}-W{WW}.md`**
```markdown
# {YYYY}-W{WW} Weekly Goals

기간: {월}일 ~ {일}일

## W1: {목표}
Parent: M1
상태: 진행중

## W2: {목표}
Parent: M2
상태: 진행중

## W3: {목표}
Parent: 유지
상태: 진행중

## 지난주 리뷰
(첫 주에는 비움)
```

### Step 6: 완료 안내

```
"운영 체계 세팅이 완료되었습니다. 4개 파일을 생성했습니다:
- goals/_foundation.md (방향, 제약, 체크인 주기)
- goals/{YYYY}.md (연간 목표)
- goals/{YYYY-MM}.md (월간 목표)
- goals/{YYYY}-W{WW}.md (주간 목표)

이제 매일 아침 /timebox를 입력하시면,
이 주간/월간 목표를 바탕으로 오늘의 일정을 제안해 드리겠습니다."
```

## Refresh 모드 (매주 월요일 등)

### 1단계: 지난주 리뷰 수집

1. `{base}/reviews/` 에서 지난주 review 파일들 Read
2. `{base}/goals/{올해}-W{지난주}.md` Read
3. 주간 목표별 달성률 계산

### 2단계: 리뷰 요약 제시

```
"지난주(W{n}) 리뷰:
W1: {목표} → {결과} {상태이모지}
W2: {목표} → {결과} {상태이모지}
W3: {목표} → {결과} {상태이모지}

월간 목표(M1: {목표}) 진행률: {n}%
```

### 3단계: 이번 주 목표 설정

```
"월간 목표 기준으로, 이번 주에 밀어야 할 것은 무엇인가요?
지난주 미완료 항목을 이월할까요?"
```

- AskUserQuestion으로 대화
- 새 주간 목표 확정 후 `{base}/goals/{YYYY}-W{WW}.md` 생성
- 월간 목표도 업데이트 필요하면 Edit

### 4단계: 월초면 월간 목표도 갱신

- 새 달 첫 주면 `{base}/goals/{YYYY-MM}.md`도 생성
- 연간 목표 대비 월간 목표 정렬 확인

## Realign 모드

Foundation부터 재검토. Cold Start와 동일하지만:
- 기존 파일을 Read하여 현재 상태 먼저 보여줌
- "어떤 부분을 바꾸고 싶으세요?" 로 시작
- 변경 부분만 Edit, 전체 재작성은 하지 않음

## 핵심 원칙

1. **먼저 많이 말하게 하기**: 질문보다 Brain-dump이 먼저
2. **질문 2~3개/턴 제한**: 질문 폭탄 금지
3. **매 턴 [핵심/빈칸/다음질문] 포맷 필수**: 사용자가 "내 말을 듣고 있다"는 느낌
4. **Parent-Child 정렬 강제**: 모든 하위 목표는 상위 목표에 연결
5. **제약 드러내기**: 용량 초과 시 즉시 경고
6. **성장/유지/회복 분리**: 세 축의 균형 확인
7. **체크인 주기 설정**: _foundation.md에 기본 주기 저장 (기본값: 5분)

## 참고

- Foundation: `{base}/goals/_foundation.md`
- 연간: `{base}/goals/{YYYY}.md`
- 월간: `{base}/goals/{YYYY-MM}.md`
- 주간: `{base}/goals/{YYYY}-W{WW}.md`
- 모든 파일은 goals/ 디렉토리에 생성
- 디렉토리가 없으면 mkdir로 자동 생성
