# /timebox-review - 패턴 리뷰

타임박스 데이터를 기간별로 분석하여 패턴을 발견하고, 시스템 개선점을 도출합니다.

## 사용법

```
/timebox-review              → 주간 리뷰 (기본)
/timebox-review weekly       → 주간 리뷰
/timebox-review monthly      → 월간 리뷰
/timebox-review yearly       → 연간 리뷰
```

## 경로
`$TIMEBOX_HOME` (미설정 시 `~/timebox`). 데이터 조회는 `timebox` CLI를 사용한다.

## 실행 흐름

> **Phase 1 (0-6단계)**: 데이터 분석 + 회고 + 리뷰 파일 저장
> **Phase 2 (7-9단계)**: 다음 기간 목표 + Foundation 갱신 + 실험 설계
> Phase 2를 실행하면 `/timebox-align` Refresh를 별도 실행할 필요 없음.

### 0단계: 데이터 수집

**기간 결정:**
- weekly: 이번 주 월~오늘 (또는 지난 주 전체)
- monthly: 이번 달 1일~오늘 (또는 지난 달 전체)
- yearly: 올해 1월~오늘 (월간 리뷰 파일을 1차 소스로 사용. 월간 리뷰가 없는 달만 daily review로 보충)

**CLI로 데이터 수집:**
1. 기간 통계:
   - weekly: `timebox stats week [--year YYYY --week WW]` → 주간 통계 JSON (Big 3 달성률, 블록 준수율, 에너지 등)
   - monthly: `timebox stats month [--year YYYY --month MM]` → 월간 통계 JSON
   - yearly: 월간 통계를 월별로 조회하거나, 월간 리뷰 파일을 직접 Read
2. 목표 데이터:
   - `timebox goals show --scope {weekly|monthly|yearly}` → 해당 스코프 목표 JSON
   - `timebox goals progress` → 주간 목표 진행률 JSON
3. carry-over: `timebox stats carry-over` → 연속 미완료 항목
4. 이전 기간 리뷰 파일 (실험 결과 평가용, CLI 미지원 → 직접 Read):
   - weekly: `{base}/reviews/{올해}-W{지난주}-weekly-review.md`
   - monthly: `{base}/reviews/{올해-지난달}-monthly-review.md`
   - yearly: `{base}/reviews/{지난해}-yearly-review.md`
5. Foundation: `{base}/goals/{올해}-foundation.md` Read (CLI 미지원)

**데이터 부족 시:** stats 결과의 days 배열이 3일 미만(weekly) / 2주 미만(monthly)이면 "기록이 부족합니다. 쌓인 데이터가 많을수록 패턴 분석이 정확해집니다."라고 안내. **억지 결론을 내리지 마라.** 있는 데이터만으로 말할 수 있는 것만 말한다.

### 1단계: 성과 요약

```
## 성과 요약

### Big 3 달성률
- 기간 내 총 Big 3 설정: {n}건
- 완료: {n}건 ({n}%)
- 진행중 carry-over: {n}건
- 드롭: {n}건

### 블록 준수율 추이
- 평균 블록 준수율: {n}%
- 최고: {날짜} {n}% / 최저: {날짜} {n}%
- 추이: {상승/하락/안정} 추세
```

### 2단계: 패턴 분석

**아래 패턴 중 데이터에서 근거가 있는 것만 분석한다. 없으면 스킵.**

**인터럽트 패턴:**
- ad-hoc 빈도: 주 {n}회, 주로 {시간대}
- 가장 흔한 인터럽트 유형: {유형}
- 인터럽트가 Big 3를 방해한 비율: {n}%
- 패턴: "{요일}에 ad-hoc이 집중됨" 등

**회피 패턴:**
- 미완료 Big 3 중 carry-over 비율: {n}%
- 3일 이상 carry-over된 항목: {리스트}
- 회피 경향 유형: "어려운 작업 뒤로 밀기" / "흥미 있는 일에 빠지기" 등

**에너지 패턴:**
- 주간 에너지 곡선: {요일별 추세}
- 최적 Deep Work 시간대: {시간대} (데이터 기반)
- 에너지 저하 시간대: {시간대}
- "점심 직후 1블록은 Shallow로 배치하면 더 나았을 것" 식의 구체적 제안

**과계획/저계획 패턴:**
- Big 3가 실제로 3개 다 끝난 날 비율: {n}%
- 평균 완료 수: {n}개/일
- 블록 추정 정확도: daily review의 `## Estimation Accuracy` 테이블에서 배율을 읽어 기간 평균을 산출
  - "추정 대비 평균 {n}배 소요. {작업 유형}이 특히 부정확" 식의 구체적 피드백
- "Big 3를 2개로 줄이고 보너스 1개를 두는 게 현실적입니다" 식의 제안

**성과 조건 패턴:**
- Big 3를 모두 달성한 날의 공통점: {조건}
- 몰입이 깊었던 블록의 공통점: {조건}
- "오전 첫 블록에 어려운 작업 → 높은 완료율" 등

### 3단계: 목표 정렬 분석

```
## 목표 정렬

### {기간} 목표 대비 진행률
- 목표 1: {목표} → {진행 상황}
- 목표 2: {목표} → {진행 상황}

### 목표 기여 분류
- 목표 직접 기여: {n}건 ({n}%)
- 유지/운영: {n}건 ({n}%)
- 예상 밖/ad-hoc: {n}건 ({n}%)
```

### 4단계: 사용자 회고

**1~3단계 분석 결과를 보여준 뒤**, 데이터에서 발견된 패턴을 짚어 질문한다.

**질문 설계 원칙:**
- 데이터에서 보인 패턴을 구체적으로 짚어서 질문 (일반론 X)
- 예: "문서 작업을 3일 연속 미뤘는데, 시간 문제인가요 시작이 어려운 건가요?"
- 예: "Big 3 달성률이 67%인데, 3개가 많은 건지 스코핑 문제인지 어떻게 느끼세요?"
- 예: "오전 몰입이 잘 되는 건 확인됐는데, 이걸 더 활용할 방법이 있을까요?"
- AskUserQuestion으로 1-2개만 물어봄, 많이 묻지 않음

**자유 회고 수용:**
- "이번 {기간}을 돌아보며 느낀 점이 있으면 자유롭게 남겨주세요"로 길게 쓸 여지를 열어둠
- 할 말이 없으면 "이번 {기간} 한줄 총평" 정도만 받아도 OK (강요하지 않음)
- **사용자 원문은 그대로 보존.** 정리, 요약, 교정하지 않는다. 날것의 기록이 가장 가치 있다.

사용자가 스킵하면 질문 없이 진행.

### 5단계: 인사이트 + 제안

**패턴 기반 실행 제안** (데이터 근거 필수):
- 구체적 행동 제안 2-3개
- 각 제안에 근거 데이터 명시
- 근거 없는 일반론 금지 ("더 열심히 하세요" 식 X)

### 6단계: 리뷰 파일 저장

**지난 실험 결과 확인:** 이전 기간 리뷰 파일(0단계에서 수집) 또는 목표 파일의 `## 이번 {기간} 실험` 섹션을 확인하여 지난 실험 결과를 평가한다. 실험이 없었으면 "첫 리뷰 — 지난 실험 없음"으로 기록.

**파일 생성 방법:**
- weekly/monthly/yearly 리뷰 파일은 Write 도구로 직접 생성 (CLI는 daily review만 지원)

**경로:**
- weekly: `{base}/reviews/{올해}-W{주차}-weekly-review.md`
- monthly: `{base}/reviews/{올해-월}-monthly-review.md`
- yearly: `{base}/reviews/{올해}-yearly-review.md`

```markdown
# {기간} Timebox Pattern Review

## 기간
{시작일} ~ {종료일}

## 지난 실험 결과
(지난 기간에 설정한 실험이 있으면 결과 평가. 없으면 "첫 리뷰 — 지난 실험 없음")
- 실험: {지난 기간 실험 내용}
- 결과: {효과 있음 / 부분 효과 / 효과 없음 / 미실행}
- 판정: {Foundation 승격 / 변형 재실험 / 드롭}

## 성과 요약
{1단계 내용}

## 패턴 분석
{2단계 내용}

## 목표 정렬
{3단계 내용}

## Reflection
{사용자가 쓴 회고 원문 그대로. 정리/요약/교정 하지 않는다.
 한 줄이면 한 줄, 한 페이지면 한 페이지.}

## Coach's Notes
{Claude 작성}
- 이번 기간 핵심 요약 (원문 + 데이터에서 뽑은 포인트)
- 패턴 연결 (이전 기간 리뷰와의 비교, 성장 궤적)
- 실험 결과 연결 (있으면)
> When-Then: {다음 기간에 [상황]이 오면, [행동]한다}
> 질문: {다음 리뷰 때 깊이 생각해볼 질문}

## 인사이트 & 제안
{5단계 내용}

## 다음 기간 목표
{7단계 내용}

## Foundation 업데이트
{8단계에서 추가/수정한 내용 요약}

## 다음 실험
{9단계 내용}
```

### 7단계: 다음 기간 목표 업데이트

리뷰 인사이트를 바탕으로 다음 기간 목표를 직접 생성한다.

**7-1. 이번 기간 목표 정산:**
- 각 목표의 최종 상태 판정: 완료 / carry-over / 드롭
- carry-over 항목: 3기간 이상 carry-over면 AskUserQuestion으로 "이월 vs 드롭" 결정 요청
- 완료 항목: 상위 목표(Parent) 진행률 업데이트 제안

**7-2. 다음 기간 목표 초안 제시:**

```
"리뷰 결과를 바탕으로 다음 {기간} 목표를 제안합니다:

{주간일 때}
- [W1] {목표} — {근거: carry-over / 월간 목표 기반 / 패턴 대응}  ← Parent: M?
- [W2] {목표} — {근거}  ← Parent: M?
- [W3] {목표} — {근거}  ← Parent: M?

{월간일 때}
- [M1] {목표} ({완료 기준}) — {근거}  ← Parent: Y?
- [M2] {목표} ({완료 기준}) — {근거}  ← Parent: Y?

이 목표들로 시작할까요? 수정하고 싶은 부분이 있으면 말씀해주세요."
```

- AskUserQuestion으로 확인
- 수정 있으면 반영 후 재확인

**7-3. 파일 생성/업데이트:**
- weekly: `{base}/goals/{올해}-W{다음주}.md` Write
- monthly: `{base}/goals/{올해-다음달}.md` Write + `{base}/goals/{올해}.md` Edit (연간 목표 상태 업데이트)
- yearly: `{base}/goals/{다음해}.md` Write + Foundation 리프레시 제안

**목표 파일 포맷 (CLI 파서 호환 필수):**
```markdown
# {기간} 목표

## Goals

- [W1] 목표 텍스트
- [W2] 목표 텍스트 (성공 기준이 있으면 괄호 안에)

## Notes

- W1 → M1, W2 → M2 (Parent 매핑)
- 지난주 핵심 인사이트 1-2줄
```
- `## Goals` 섹션의 `- [ID] 텍스트 (기준)` 포맷이 CLI가 파싱하는 영역
- `## Notes` 섹션은 CLI가 무시하지만 Claude가 읽는 메타데이터 (Parent 매핑, 맥락)

> **Note:** review의 목표 설정은 데이터 기반 자연 갱신입니다 (주간/월간 전환기의 정기 회고). `/timebox-align`은 방향 전환(이직, 큰 목표 변경 등) 시 사용자가 직접 실행하여 입력 기반으로 재정렬합니다. review에서 이미 목표를 세웠으면 align Refresh는 확인만 합니다.

### 8단계: Foundation 자기 지식 업데이트

리뷰에서 발견된 **반복 패턴**을 `{base}/goals/{올해}-foundation.md`의 `## 자기 지식` 섹션에 축적한다.

**업데이트 기준:**
- 2회 이상 반복 확인된 패턴 → 자기 지식에 추가
- 기존 항목과 일치하는 새 데이터 → 확인 횟수/기간 업데이트
- 지난 실험에서 효과 확인된 전략 → "검증된 전략"에 승격
- 더 이상 유효하지 않은 항목 → 삭제 또는 수정

**업데이트 방식:**
1. `{base}/goals/{올해}-foundation.md` Read
2. `## 자기 지식` 섹션 확인
3. 이번 리뷰에서 발견된 패턴과 기존 항목 비교
4. Edit으로 추가/수정 (절대 다른 섹션 건드리지 않음)

**자기 지식 섹션 구조:**
```markdown
## 자기 지식

### 에너지 패턴
- {패턴}: {근거 — 확인 기간/횟수}

### 몰입 조건
- {조건}: {근거}

### 회피 트리거
- {트리거}: {근거}

### 추정 정확도
- 전체 평균: {n}배 (기간: {확인 기간})
- {작업 유형별}: {n}배 (예: 리팩토링 1.8x, 신규 개발 2.3x)

### 검증된 전략
- {전략}: {실험 기간} → {결과} (출처: {W?? 리뷰})
```

**성장 로그:**
- daily review의 Reflection과 Coach's Notes에서 핵심 자각/깨달음을 Foundation `## 성장 로그`에 누적
- 월간/연간 리뷰 때 성장 로그를 통합 정리

```markdown
## 성장 로그
- {YYYY-MM-DD}: {핵심 자각} (출처: {리뷰})
```

### 9단계: 다음 기간 실험 설계

패턴 분석(2단계)과 인사이트(5단계)에서 도출된 **구체적 행동 변화 1개**를 "실험"으로 설정한다.

**실험 설계 원칙:**
- 한 기간에 실험 1개만 (여러 개 동시에 바꾸면 뭐가 효과 있었는지 모름)
- 구체적이고 관찰 가능해야 함 ("더 열심히" X → "출근 후 15분 내 헤드폰 착용" O)
- 성공 기준 명시 ("5일 중 3일 이상 실행하면 성공")

**실험 포맷:**
```
이번 {기간} 실험:
- 실험: {구체적 행동}
- 근거: {어떤 패턴에서 도출했는지}
- 성공 기준: {관찰 가능한 기준}
- 기록 방법: {timebox-end 시 실험 실행 여부 체크}
```

- AskUserQuestion: "이 실험으로 진행할까요? 다른 걸 시도하고 싶으면 말씀해주세요."
- 확정된 실험을 7단계에서 생성한 주간/월간 목표 파일의 `## 이번 {기간} 실험` 섹션에 추가 (Edit)

**실험 결과 → Foundation 연동:**
- 다음 리뷰(6단계 리뷰 파일)의 `## 지난 실험 결과`에서 평가
- 효과 있음 → 8단계에서 Foundation `자기 지식 > 검증된 전략`에 승격
- 부분 효과 → 변형하여 재실험
- 효과 없음/미실행 → 드롭 또는 원인 분석 후 재설계

### 10단계: 완료 안내

```
"리뷰가 완료되었습니다.

📄 리뷰 파일: {경로}
```

## 참고

- 주간 통계: `timebox stats week [--year YYYY --week WW]`
- 월간 통계: `timebox stats month [--year YYYY --month MM]`
- 목표 조회: `timebox goals show --scope {weekly|monthly|yearly}`
- 목표 진행률: `timebox goals progress`
- carry-over: `timebox stats carry-over`
- foundation.md, 이전 기간 리뷰 파일은 직접 Read (CLI 미지원)
- 리뷰 파일(weekly/monthly/yearly)은 Write로 직접 생성 (CLI는 daily review만 지원)
- 데이터가 부족하면 억지 결론을 내리지 않는다. "데이터 부족"이라고 솔직히 말한다.
- 수치는 CLI stats 결과(JSON)를 사용한다. 직접 세거나 추정하지 않는다.
- 기존 timebox-end의 7단계(주간 목표 체크)는 이 커맨드로 대체된다.
- **목표 파일 생성은 Write, 기존 목표 파일 수정은 Read 후 Edit.**
- **Foundation 수정은 반드시 Read 후 Edit. 자기 지식/성장 로그 섹션만 수정하고 다른 섹션은 절대 건드리지 않는다.**
- 7-9단계는 데이터 부족 경고가 떠도 실행한다. 패턴이 약해도 목표 설정과 실험은 할 수 있다.
- **사용자 회고 원문은 절대 수정/정리하지 않는다.** 날것의 기록이 가장 가치 있다.
- Coach's Notes가 요약 기능을 겸하므로 별도 요약 섹션은 두지 않는다.
- Coach's Notes 끝에는 반드시 When-Then 1개 + 질문 1개를 포함한다.
