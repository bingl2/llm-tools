# /timebox-review - 패턴 리뷰

타임박스 데이터를 기간별로 분석하여 패턴을 발견하고, 시스템 개선점을 도출합니다.

## 사용법

```
/timebox-review              → 주간 리뷰 (기본)
/timebox-review weekly       → 주간 리뷰
/timebox-review monthly      → 월간 리뷰
/timebox-review yearly       → 연간 리뷰
```

## 경로 설정

```
Base Path: $TIMEBOX_HOME (환경변수 미설정 시 ~/timebox)
Plans: {base}/plans/
Logs: {base}/logs/
Reviews: {base}/reviews/
Goals: {base}/goals/
```

- 시작 시 `echo $TIMEBOX_HOME`으로 확인. 없으면 `~/timebox` 사용.

## 실행 흐름

### 0단계: 데이터 수집

**기간 결정:**
- weekly: 이번 주 월~오늘 (또는 지난 주 전체)
- monthly: 이번 달 1일~오늘 (또는 지난 달 전체)
- yearly: 올해 1월~오늘

**읽을 파일:**
1. 해당 기간의 daily review 파일들:
   - Glob 패턴: `{base}/reviews/{YYYY-MM-DD}-timebox-review.md`
   - 기간에 해당하는 날짜만 필터
2. 해당 기간의 로그 파일들:
   - Glob 패턴: `{base}/logs/{YYYY-MM-DD}/*.md`
3. 목표 파일:
   - weekly: `{base}/goals/{올해}-W{해당주}.md`
   - monthly: `{base}/goals/{올해-해당월}.md`
   - yearly: `{base}/goals/{올해}-foundation.md`, `{base}/goals/{올해}-yearly.md`

**데이터 부족 시:** 리뷰 파일이 3일 미만(weekly) / 2주 미만(monthly)이면 "기록이 부족합니다. 쌓인 데이터가 많을수록 패턴 분석이 정확해집니다."라고 안내. **억지 결론을 내리지 마라.** 있는 데이터만으로 말할 수 있는 것만 말한다.

### 1단계: 성과 요약

```
## 성과 요약

### Big 3 달성률
- 기간 내 총 Big 3 설정: {n}건
- 완료: {n}건 ({n}%)
- 진행중 carry-over: {n}건
- 드롭: {n}건

### 블록 준수율 추이
- 평균 블록 준수율: {n}%
- 최고: {날짜} {n}% / 최저: {날짜} {n}%
- 추이: {상승/하락/안정} 추세
```

### 2단계: 패턴 분석

**아래 패턴 중 데이터에서 근거가 있는 것만 분석한다. 없으면 스킵.**

**인터럽트 패턴:**
- ad-hoc 빈도: 주 {n}회, 주로 {시간대}
- 가장 흔한 인터럽트 유형: {유형}
- 인터럽트가 Big 3를 방해한 비율: {n}%
- 패턴: "{요일}에 ad-hoc이 집중됨" 등

**회피 패턴:**
- 미완료 Big 3 중 carry-over 비율: {n}%
- 3일 이상 carry-over된 항목: {리스트}
- 회피 경향 유형: "어려운 작업 뒤로 밀기" / "흥미 있는 일에 빠지기" 등

**에너지 패턴:**
- 주간 에너지 곡선: {요일별 추세}
- 최적 Deep Work 시간대: {시간대} (데이터 기반)
- 에너지 저하 시간대: {시간대}
- "점심 직후 1블록은 Shallow로 배치하면 더 나았을 것" 식의 구체적 제안

**과계획/저계획 패턴:**
- Big 3가 실제로 3개 다 끝난 날 비율: {n}%
- 평균 완료 수: {n}개/일
- "Big 3를 2개로 줄이고 보너스 1개를 두는 게 현실적입니다" 식의 제안

**성과 조건 패턴:**
- Big 3를 모두 달성한 날의 공통점: {조건}
- 몰입이 깊었던 블록의 공통점: {조건}
- "오전 첫 블록에 어려운 작업 → 높은 완료율" 등

### 3단계: 목표 정렬 분석

```
## 목표 정렬

### {기간} 목표 대비 진행률
- 목표 1: {목표} → {진행 상황}
- 목표 2: {목표} → {진행 상황}

### 목표 기여 분류
- 목표 직접 기여: {n}건 ({n}%)
- 유지/운영: {n}건 ({n}%)
- 예상 밖/ad-hoc: {n}건 ({n}%)
```

### 4단계: 사용자 회고

AskUserQuestion으로 1-2개 질문:
- "이 기간 동안 가장 의미 있었던 성과는 무엇인가요?"
- "다음 기간에 바꾸고 싶은 것이 있나요?"

사용자가 스킵하면 질문 없이 진행.

### 5단계: 인사이트 + 제안

**패턴 기반 실행 제안** (데이터 근거 필수):
- 구체적 행동 제안 2-3개
- 각 제안에 근거 데이터 명시
- 근거 없는 일반론 금지 ("더 열심히 하세요" 식 X)

### 6단계: 리뷰 파일 저장

**경로:**
- weekly: `{base}/reviews/{올해}-W{주차}-weekly-review.md`
- monthly: `{base}/reviews/{올해-월}-monthly-review.md`
- yearly: `{base}/reviews/{올해}-yearly-review.md`

```markdown
# {기간} Timebox Pattern Review

## 기간
{시작일} ~ {종료일}

## 성과 요약
{1단계 내용}

## 패턴 분석
{2단계 내용}

## 목표 정렬
{3단계 내용}

## 사용자 회고
{4단계 내용}

## 인사이트 & 제안
{5단계 내용}
```

## 참고

- 데이터가 부족하면 억지 결론을 내리지 않는다. "데이터 부족"이라고 솔직히 말한다.
- 수치는 반드시 실제 파일에서 세어서 계산한다. 추정/감 금지.
- 기존 timebox-end의 7단계(주간 목표 체크)는 이 커맨드로 대체된다.
- **리뷰 파일은 Write로 생성. 기존 파일이 있으면 Read 후 Write로 덮어쓰기.**
