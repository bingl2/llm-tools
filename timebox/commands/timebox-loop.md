# /timebox-loop - 자동 블록 알림 (loop 연동)

`/loop {체크인주기}m /timebox-loop` 으로 사용 (주기는 `_config.md`의 `checkin_interval`, 기본 15분).
질문 없이 현재 상태만 알려주고 끝. 사용자 응답을 기다리지 않는다 (AskUserQuestion 사용 금지).

## 경로
`$TIMEBOX_HOME` (미설정 시 `~/timebox`). 마스터 파일: `{base}/plans/{YYYY-MM-DD}.md` (읽기 전용).

## 실행 흐름

### 1단계: 상태 확인

1. `date +%H:%M`으로 현재 시각 확인
2. 오늘 타임박스 파일 Read
   - 경로: `{base}/plans/{오늘날짜}.md`
   - **파일 없으면**: 아무것도 출력하지 않고 종료 (loop에서 노이즈 방지)
3. 주간 목표 파일 Read (있으면): `{base}/goals/{올해}-W{이번주}.md` — 코치 넛지의 목표 연결용
4. Schedule 파싱하여 현재 블록 판단
5. 오늘 로그 중 최신 1개 Read (있으면): `{base}/logs/{오늘날짜}/` 에서 가장 최근 파일 — 방향 이탈 감지용

### 2단계: 상황별 알림 (텍스트 출력만)

**블록 진행 중 (항상 현재 상태 표시):**
```
📌 {Big 3 항목} — 남은 {n}분
   → {다음 미완료 체크리스트 항목 1개}
   {블록타입} {HH:MM}-{HH:MM} | 다음: {다음 블록타입}
```
- **Content-first**: 첫 줄은 "지금 뭘 해야 하는지" — 한 줄만 읽어도 흐름 복귀 가능
- 두번째 줄은 "지금 바로 할 것" 1개만 — 체크리스트 전체가 아닌 다음 액션
- 블록 타입/시간은 부가 정보로 세번째 줄에 배치
- Big 3와 무관한 작업 중이면 `/timebox-log`로 기록하거나, 원래 작업으로 돌아올 것을 안내

**블록 종료 5분 전:**
```
⏰ 5분 남음 — 지금 하던 것 매듭짓기
   다음: {다음 블록타입} — {다음 Focus}
```

**블록 전환 시점 (Break 진입):**
```
🔔 블록 전환!
   완료: {이전 블록타입} ({HH:MM}-{HH:MM})
   지금: Break — 자리에서 일어나세요. 물 한 잔.
   다음: {HH:MM} | {다음 블록타입} — {Focus}
```

**Break 끝나고 새 블록 시작:**
```
🟢 {Big 3 항목 또는 Shallow 내용} 시작! ({n}분)
   첫 번째 할 일: {미완료 체크리스트 첫 항목}
```

**블록 시간 초과 (예정 종료 시각을 넘긴 경우):**
```
⚠️ {블록타입} 시간이 {n}분 초과됐습니다. 멈추세요.
   "조금만 더"는 다음 블록을 잡아먹습니다.
   다음: {HH:MM} | {다음 블록타입}
```

**근무 시간 종료:**
```
🏁 업무 시간 끝.
   Big 3: {완료}/{전체} — `/timebox-end`로 마감하세요.
```

### 3단계: 코치 한마디 (명언/동기부여)

상태 알림 바로 아래에 짧은 동기부여 문구를 1개 덧붙인다.

**4가지 스타일을 비중에 따라 섞어 사용:**

1. **상황 인식형 넛지 (50%)** — 현재 작업의 "왜"를 환기하여 흐름 복귀 유도
   ```
   🎯 {현재 작업}의 핵심은 {한 문장 요약}. 거기로 돌아가.
   ```
   - 현재 Focus 작업이 주간/월간 목표와 어떻게 연결되는지 짧게 환기
   - 예: `🎯 API 리팩토링의 핵심은 응답속도 50% 개선. 거기에 집중.`
   - 예: `🎯 지금 하는 건 이번 주 목표 "TNA-5967 완료"의 일부. 계속 가.`

2. **작업 맞춤형 창작 (20%)** — 현재 Focus 작업에 의미를 부여하는 성취감 자극
   ```
   💡 {현재 작업과 연결된 창작 문구}
   ```

3. **거장의 지혜 (15%)** — 몰입, 장인정신, 끈기, 시간 관리 관련 실제 명언
   ```
   💬 "{명언}" — {이름}
   ```

4. **팩폭 스타일 (15%)** — 남은 시간의 귀중함을 직설적으로 환기 (느슨해질 때)
   ```
   🔥 {뼈 때리는 한마디}
   ```

**규칙:**
- **반드시 한글로 작성** (영어 명언도 한글로 번역하여 출력)
- 매번 다른 문구 사용 (반복 금지)
- 1~2문장, 짧고 강렬하게
- 실제 명언 인용 시 반드시 출처(이름) 명시
- 현재 Focus 작업, 남은 시간, 블록 상황에 맞춰 선택
- **기본**: 상황 인식형 넛지 우선 (흐름 복귀가 최우선 목적)
- 남은 시간 많을 때: 몰입/장인정신 강조
- 남은 시간 적을 때: 마무리/집중력 강조
- 시간 초과 시: 팩폭 스타일 우선

### 4단계: 방향 이탈 감지 (간략 버전)

- 현재 블록이 Deep Work인데, 마지막 `/timebox-log`가 Big 3와 무관했으면:
```
📍 지금은 Deep Work 시간입니다. Big 3에 집중하고 있나요?
```
- 이 메시지는 같은 블록 내에서 1회만 출력 (반복 방지)

## 핵심 규칙

- **절대 AskUserQuestion 사용 금지** — loop에서 돌기 때문에 질문하면 멈춤
- **조용한 게 기본** — 알릴 게 없으면 출력 없음
- **짧게** — 최대 3줄. 길면 읽지 않음
- **코치 톤 유지** — "멈추세요", "집중하고 있나요?" (부드럽지만 직접적)

## 참고

- 체크인 주기는 `/timebox`에서 설정 (기본값: _config.md의 checkin_interval)
- 마스터 파일: `{base}/plans/{YYYY-MM-DD}.md`
- 마스터 파일 수정하지 않음 (읽기 전용)
