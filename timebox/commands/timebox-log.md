# /timebox-log - 블록 작업 로그 + 체크리스트 갱신

현재 세션에서 수행한 작업을 타임박스 로그로 남기고, 마스터 파일의 체크리스트도 함께 갱신합니다.

## 경로
`$TIMEBOX_HOME` (미설정 시 `~/timebox`). 로그: `{base}/logs/{YYYY-MM-DD}/` (없으면 자동 생성). 마스터 파일은 Read → Edit만 (Write 금지).

## 실행 흐름

### 1단계: 현재 상태 확인

1. `date +%H:%M`으로 현재 시간 확인
2. 오늘 타임박스 파일 Read
   - 경로: `{base}/plans/{오늘날짜}.md`
   - **파일 없으면**: "`/timebox`로 먼저 플랜을 생성해주세요." 안내 후 종료
3. 현재 시각이 어느 블록에 해당하는지 파악

### 2단계: 작업 내용 정리

현재 세션의 대화 내용을 기반으로:
1. 어떤 작업을 했는지 요약
2. Big 3 중 어떤 항목과 관련되는지 매칭
3. 완료된 작업과 진행 중인 작업 분류
4. **이벤트 분류**: 이 블록의 흐름 유형 판단
   - `deep-work`: 계획대로 Deep Work 진행
   - `interrupt`: 외부 인터럽트로 계획 변경 (누구에게서, 무엇 때문에)
   - `switch`: 자발적 작업 전환 (왜 전환했는지)
   - `ad-hoc`: 예상 외 작업 발생
   - `break`: 휴식/이동
   - `shallow`: Shallow Work (이메일, 리뷰, 미팅 등)
5. **목표 연결** (`related_to`): 이 작업이 어떤 목표에 기여하는지
   - Big 3 항목: `Big 1`, `Big 2`, `Big 3` (마스터 파일의 Big 3 번호와 1:1 매칭)
   - 주간/월간 목표 (예: "W1: TNA-5967 완료")
   - 없으면: "ad-hoc" 또는 "유지/운영"
6. **에너지 레벨 확인** (AskUserQuestion, 스킵 가능):
   - "지금 에너지 레벨은? (1-5, 스킵 가능)"
   - 에너지 = **지금 집중할 수 있는 상태**: 1 거의 못함, 2 낮음, 3 보통, 4 좋음, 5 최고 집중
   - 스킵하면 energy 필드를 비운다
   - 축적되면 최적 Deep Work 시간대를 찾는 데 활용됨 (Foundation 에너지 패턴)

### 3단계: 로그 파일 생성/업데이트

**경로**: `{base}/logs/{YYYY-MM-DD}/{HHmm}-timebox-{블록타입}.md`

```markdown
# {HH:MM} Timebox Log - {블록타입}

## Block
{HH:MM}-{HH:MM} | {블록타입}
Focus: {Big 3 항목}

## Event
- type: {deep-work | interrupt | switch | ad-hoc | break | shallow}
- related_to: {Big 3 항목 번호 또는 목표, 없으면 "ad-hoc"}
- energy: {1-5, 체감 에너지 수준}
- trigger: {interrupt/switch일 경우 원인. 없으면 생략}

## Work Done
- {작업 내용 1}
- {작업 내용 2}

## Status
- Big 3 #1: {진행률 또는 완료}
- Big 3 #2: {진행률}

## Decisions
- {주요 결정사항}

## Notes
- {메모}
```

### 4단계: 마스터 파일 갱신

1. 마스터 파일을 Read하여 최신 내용 확인
2. 완료된 작업: `- [ ]` → `- [x]` 변환 (Edit)
3. Big 3 완료 시: `1. [ ]` → `1. [x]` 변환 (Edit)
4. 새로운 서브태스크 추가 (필요시 Edit)
5. Energy Log 테이블에 현재 시각 + energy 값 추가 (Edit):
   `| {HH:MM} | {energy}/5 | {간단 메모} |`
6. **반드시 Read 후 Edit 사용. Write 금지.**

### 5단계: 진행률 안내 + 코치 피드백

#### A. 진행률

```
## 타임박스 진행률

Big 3: 1/3 완료
- [x] API 리팩토링
- [ ] 프론트엔드 수정 (60% 진행)
- [ ] 테스트 작성

현재 블록: Deep Work (14:00-15:30) - 남은 시간 45분
다음 블록: Break (15:30-15:45)
```

**Big 3 정렬도:**
- 이 세션 작업이 Big 3 중 어디에 기여했는지 명시
- 기여도 없으면 솔직하게: "이번 작업은 Big 3와 직접 관련이 없습니다."
- 하루 전체 시간 중 Big 3에 투자한 비율 계산:
  `"오늘 Deep Work {n}시간 중 Big 3에 실제 투자: {n}시간 ({n}%)"`

#### B. 코치 피드백

로그의 이벤트 타입, 에너지 레벨, 하루 맥락을 종합하여 **상황에 맞는 짧은 코칭 메시지**를 생성한다.

**출력 형식:**
```
---
💬 코치: {상황 인식 메시지}
{선택적: 마인드풀니스 큐}
→ 다음 액션: {구체적 다음 할 일 1개}
```

**이벤트 타입별 코칭 전략:**

| 이벤트 타입 | 코칭 방향 | 예시 |
|---|---|---|
| `deep-work` (순조) | 격려 + 진행 위치 | "Big 3의 2/3 지점이야. 페이스 유지." |
| `deep-work` (완료) | 성취 인정 + 다음 유도 | "API 리팩토링 끝. 오늘 Big 3 중 하나 해치웠다. 다음 타겟으로." |
| `interrupt` | 수용 + 리셋 가이드 | "인터럽트는 일어나는 거야. 숨 한번 쉬고, 아까 하던 {작업}으로 돌아가." |
| `switch` | 전환 이유 인정 + 원래 작업 리마인드 | "전환 이유가 있었겠지. 기록했으니 됐다. 원래 계획: {Big 3 항목}." |
| `ad-hoc` | 현실 인정 + Big 3 리마인드 | "예상 외 일은 생기는 법이야. 처리했으면 Big 3로 복귀." |
| `break` | 리프레시 격려 | "쉬는 것도 일의 일부. 제대로 쉬어." |
| `shallow` | 중립 + 시간 인식 | "Shallow는 필요하지만 시간제한이 핵심. 남은 Deep Work: {n}시간." |

**에너지 레벨 연동 (energy 필드 활용):**

| 에너지 | 추가 코칭 |
|---|---|
| 1-2 (저) | "에너지가 바닥이네. 5분만 눈 감거나 스트레칭. 억지로 밀어붙이면 효율만 떨어져." |
| 3 (보통) | 기본 이벤트 메시지만 |
| 4-5 (고) | "컨디션 좋을 때 어려운 걸 해야지. 지금이 그 타이밍." |

**하루 전체 맥락 반영:**

오늘 플랜 대비 현재 위치를 한 줄로:
- 순조로울 때: "오늘 계획의 {n}% 지점. 잘 나가고 있어."
- 밀릴 때: "계획보다 좀 밀리고 있지만, 남은 {n}시간이면 충분해. 우선순위만 지켜."
- 거의 다 했을 때: "Big 3 거의 다 왔다. 마지막 하나만 마무리하면 오늘 승리."
- 시간 초과 중: "운동 많이 된다. 스트레스 받을 거야. 근데 그게 성장이야."

**마인드풀니스 큐 (조건부):**

interrupt/switch 이벤트이거나 energy ≤ 2일 때만 출력. 아래 중 하나를 랜덤 선택:
```
🧘 잠깐. 숨 3번만 깊게 쉬고 시작하자.
```
- "숨 3번 깊게"
- "10초 눈 감기"
- "어깨 힘 빼기"
- "지금 이 순간에만 집중"
- "완벽하지 않아도 된다. 시작만 해"

**규칙:**
- 코칭 메시지는 반드시 한글
- 이벤트 타입 + 에너지 + 하루 맥락 세 축을 모두 고려하여 개인화
- generic한 멘트 금지 — 구체적 작업명, 구체적 숫자를 넣어 말하기
- 마인드풀니스 큐는 해당 조건에서만 (매번 나오면 노이즈)

## 참고

- 마스터 파일: `{base}/plans/{YYYY-MM-DD}.md`
- 로그 경로: `{base}/logs/{YYYY-MM-DD}/`
- 날짜별 로그 폴더가 없으면 자동 생성
- 프로젝트명 축약: mrt-experiences → mrt-exp 등
- **마스터 파일은 절대 Write 금지. 반드시 Read 후 Edit만 사용.**
